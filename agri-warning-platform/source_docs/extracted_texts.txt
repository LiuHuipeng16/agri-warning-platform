P0: 一项目简介
P1: 随着农业生产规模不断扩大以及气候环境变化加剧，病虫害对农业生产安全的影响日益突出。传统病虫害防治方式主要依赖人工巡查与经验判断，存在预警不及时、风险分析能力不足以及信息获取效率低等问题，难以满足智慧农业背景下对农业风险智能分析与辅助决策的需求。
P3: 针对上述问题，本项目设计并实现了一套基于RAG与气象风险分析的农业病虫害智能预警系统。系统融合实时气象数据、农业病虫害规则模型、农业知识库以及人工智能技术，实现病虫害风险分析、智能预警、农业知识问答以及AI辅助决策等功能。
P5: 系统首先通过 Open-Meteo API 获取实时天气与未来天气数据，并对小时级气象数据进行聚合处理，形成温度、湿度、降水量以及风速等农业风险分析指标。随后系统结合农业专家预警规则，对病虫害发生环境进行区间匹配与风险分析，并通过风险评分机制综合计算病虫害风险指数，从而实现病虫害预警的自动生成。
P7: 为了增强系统的智能化与专业化能力，本项目进一步引入基于RAG（Retrieval-Augmented Generation）架构的农业智能问答模块。系统通过向量化检索农业知识库内容，并结合用户问题、业务上下文以及系统提示词构建Prompt，实现农业知识增强问答，从而有效降低大语言模型幻觉问题，提高AI回答的专业性与准确性。
P9: 此外，系统还实现了图文与环境融合分析能力。在AI图片识别过程中，系统不仅能够分析病虫害图像信息，还能够结合当前天气、未来天气、病虫害适宜环境条件、风险预警信息以及农业知识库内容进行综合分析，实现面向农业场景的智能化辅助决策。
P11: 本项目探索了气象数据分析、农业规则引擎、知识增强生成以及人工智能技术在智慧农业领域中的融合应用，为农业病虫害风险分析与农业数字化建设提供了一种可行的技术方案。
P14: 二、项目研究内容与实施的基础条件
P15: 本项目围绕智慧农业背景下病虫害风险分析与农业智能辅助决策需求展开研究，重点研究气象数据分析、农业规则引擎、风险评分机制、知识增强生成以及多模态AI融合分析等内容，构建一套面向农业场景的病虫害智能预警系统。
P17: ## （一）项目研究内容
P19: ### 1. 气象数据驱动的病虫害风险分析
P21: 系统通过 Open-Meteo API 获取实时天气与未来天气数据，包括温度、湿度、降水量以及风速等信息。由于天气接口返回的是小时级数据，因此系统会对天气数据进行聚合处理，形成平均湿度、累计降水量、最大风速以及温度区间等农业分析指标。
P23: 在此基础上，系统结合病虫害适宜发生环境条件以及农业专家规则，对天气数据进行区间匹配分析，从而实现病虫害风险判断。
P25: ---
P27: ### 2. 基于规则引擎的病虫害预警机制
P29: 系统设计了一套农业病虫害预警规则模型。每条规则可配置温度、湿度、降水量以及风速等条件范围，并支持：
P31: * 区间匹配
P32: * 最小值匹配
P33: * 最大值匹配
P34: * 条件忽略
P36: 系统通过规则匹配机制对天气指标与规则条件进行分析，并进一步结合风险评分模型计算病虫害风险指数。
P38: 系统并非简单采用“命中即预警”的方式，而是通过多气象因子综合计算风险指数，实现更加合理的病虫害风险分析。
P40: ---
P42: ### 3. 可解释型风险评分机制
P44: 为了提高病虫害预警结果的透明性与可信度，系统设计了风险评分与命中依据分析机制。
P46: 系统根据湿度、温度、降水量以及风速四类环境因素，对病虫害风险进行综合评分，并生成风险贡献明细。例如：
P48: * 湿度贡献分
P49: * 温度贡献分
P50: * 降水贡献分
P51: * 风速贡献分
P53: 同时系统还能够记录各环境指标与规则条件的匹配情况，从而实现病虫害预警结果的可解释分析。
P55: 该机制能够帮助用户理解“为什么会触发预警”，提高系统分析结果的可信度。
P57: ---

R0: 1．项目简介 / 随着我国的快速发展，人民对渔业产品的需求日益增长，传统的养殖方式已难以满足当下需求。同时，受原料成本、商品供求情况、市场竞争、货币价值情况、国家政策导向等多种因素影响，水产品的价格和质量波动频繁，不仅给养殖户和企业带来经营风险，也引发了一系列社会问题，如市场秩序不稳定、消费者信任度降低等。与此同时，随着人民生活水平的日益提高，食品安全问题愈发成为消费者关注的重中之重。在水产养殖领域，缺乏有效的质量追溯体系，使得消费者难以了解水产品的来源和养殖过程，对产品质量安全存在诸多疑虑，这在很大程度上制约了水产养殖业的健康发展。 /     为解决上述问题，本项目致力于构建一套先进的水产溯源养殖系统。该系统运用物联网、大数据、区块链等前沿技术，实现对水产养殖全过程的精准监测与数据采集，包括水质、饲料、鱼苗、病害防治等关键环节。通过建立完善的追溯链条，为每一批次的水产品赋予唯一的身份标识，消费者只需扫描产品二维码，即可获取该水产品从养殖源头到餐桌的详细信息，如养殖水域环境、养殖周期、用药情况、运输过程等，从而实现对消费者透明的质量管控。这不仅能有效保障水产品的质量安全，提升消费者的信任度和满意度，还能帮助养殖户和企业优化养殖管理流程，降低生产成本，提高市场竞争力，促进水产养殖业的可持续发展，同时也为政府监管部门提供了高效便捷的监管工具，助力打造安全、有序、健康的水产养殖市场环境。 /     在技术实现方面，本系统基于功能强大且高效稳定的 Java 面向对象编程语言进行开发，并选用业界领先的 IntelliJ IDEA 作为集成开发环境，借助 Vue 框架和 ECharts 工具来实现系统丰富且直观的可视化功能，从而确保用户能够以最为便捷、高效的方式获取和解读各类关键信息。经过多次严谨、科学的实验验证，本系统在每一次的测试过程中均能够稳定、准确地查询到完整且详实的溯源信息，充分证明了系统的可靠性和有效性，为其在实际生产生活中的广泛应用奠定了坚实的基础
R1: 2．本项目目前国内外研究现状 / 在我国，近年来产品溯源得到了越来越广泛的应用和推广。在食品领域，众多企业已开始尝试构建溯源体系，尤其是一些大型的肉类、果蔬加工企业，通过采用二维码、RFID 等技术手段，实现了从农田到餐桌、从养殖场到超市货架的部分信息追溯，一定程度上提升了消费者对产品质量安全的信心。然而，在水产养殖方面，溯源系统的建设仍处于起步阶段，虽然部分大型水产企业开展了一些试点工作，但整体的覆盖率较低，且现有的溯源系统大多存在信息不完整、数据更新不及时、追溯环节有限等问题，难以满足消费者对水产品全方位精准溯源的需求。 / 从国际上看，一些发达国家在产品溯源领域起步较早且技术较为成熟。例如欧盟国家在农产品和食品溯源方面制定了严格的法规和标准，利用先进的物联网技术实现了对各类产品的精细化追溯，其溯源系统不仅涵盖生产环节，还延伸至加工、运输、销售等全链条，消费者能够获取到详尽且准确的产品信息，有效保障了产品质量安全和市场秩序。美国则在大数据和区块链技术应用于产品溯源上处于领先地位，通过整合各方数据资源，构建起高度可信且高效的溯源平台，实现了对不同行业产品的深度追溯和质量管控，为企业优化生产流程、政府实施精准监管以及消费者放心消费提供了有力支撑。但即使在国际上，水产溯源养殖系统的发展也面临着如不同国家和地区标准不统一、数据共享困难、系统建设和运维成本高等诸多挑战，这也为我国的水产溯源养殖系统研发提供了宝贵的经验借鉴和改进方向，促使我们在借鉴国际先进技术的同时，结合国内实际情况，探索出一条适合我国国情的水产溯源养殖系统发展之路。 / 
R2: 本项目研究的目的 / 本项目致力于打造一套能够确保精准溯源的水产智慧养殖溯源系统。借助先进的电子标签设备，为每一批次的水产品赋予独一无二的标识，如同为其配备一张 “电子身份证”。养殖户可以依托该系统对水产养殖过程中产生的各类数据进行深度分析，涵盖水质变化、饲料投喂量、鱼苗生长状况以及病害发生情况等关键信息，进而依据分析结果灵活调整养殖策略，有效提升养殖效益，并有力保障水产品的质量安全。 / 凭借水产品上的电子标签，监管者和消费者能够轻松查询到该产品从育苗、养殖到捕捞、加工等全生命周期的详细信息，诸如育苗的时间和地点、养殖水域的环境参数、养殖过程中的操作记录、捕捞的日期、检疫检验的具体情况等，以此确保流向市场的每一份水产品都具备可靠的信息源头、优质的品质保障，既方便监管部门实施严格且高效的监督管理，又能够显著增强从生产源头到销售终端整个过程中的管理效能以及质量溯源水平，为水产养殖业的健康、可持续发展筑牢坚实基础，满足消费者对于水产品质量安全的高要求与殷切期望，同时也为行业的规范化、智能化发展提供有力的技术支撑与保障手段。

/ （1）基于加权最小二乘法的同类传感器数据融合 /    	水产环境参数关系复杂，不同的传感器采集的数据类型不同，信息处理的方 / 式不同，数据存储的格式也不同，致使数据很难通过简单处理实现数据融合。 / 同类传感器数据级融合是将表示某一环境特征的同类传感器的多个测量数据进行融合，当其中某个传感器出现异常，其他的传感器可以及时修正。这样即使单个传感器出现故障，整个系统的监测功能也不会受到影响。异类传感器决策级融合是将异类传感器采集到的表示不同环境特征的数据进行融合。为了提高测量数据的精度，采用加权最小二乘法对来自不同节点的同类传感器进行数据融合; 第 2 层是决策级融合，将经过第 1 层数据融合的数据利用神经网络构建几种环境特征参数对目标区域环境状况进行综合评判，最终获得水产养殖环境分析结果。基于加权最小二乘法同类传感器数据融合计算流程如图8所示。 /                       / 图9 基于加权最小二乘法同类传感器数据融合计算流程图 / 基于 ARIMA算法的模型 /     ARIMA模型全称差分整合移动平均自回归模型（Autoregressive Integrated Moving Average mod‐el），简称 ARIMA 模型。该模型是对时间序列数据进行分析和预测比较完善和精确的算法，模型十分简单，只需要内生变量而不需要借助其他外生变量。 / 基于随机森林算法的模型 / 随机森林（Random Forest，RFR）是将决策树中的 CART 算法和 Bagging 算法相结合的一种新算法。随机森林可以用于分裂，也可以用于回归。随机森林可以处理高纬度的数据，训练之前不需要特意地做特征选择，而且可以建立很多决策树，预防了过拟合风险。  / 
R1: 2  研究项目的创新点 / 本项目是基于区块链技术的水产溯源管理系统，通过大数据等相关技术，实现水产信息的自动采集并支持溯源操作。其创新点包括： / 一鱼一码一市场，追溯粒度精细； / 溯源信息全自动采集，避免了人工录入信息的不可靠性和不及时性； / 标识设备采用北斗卫星定位及电子范围辐射设计，防止水产养殖过程中的财产丢失； / (4) 无须额外包装，降低管理成本； / (5) 第一标签可循环使用，进一步降低成本；⑥第二标签采用食品级材料，保障食品安全； / (6) 适应当前水产交易的多样化方式，具备较强的推广应用价值。   / 2.1 存储模式创新 /     项目采用水产智慧养殖区块溯源“链上链下”双模式存储策略，构建水产智慧养殖区块溯源链的双模式存储云数据库和资源池，实现“链上链下”存储内容的协作与交换，支持虚拟资源的高效调度与管理以及跨数据中心的云存储。数据库分为多类子库，全面记录每一条水产品从孵化、养殖到销售的全过程，并在销售完成一个月后自动删除相关数据，释放数据库存储空间，为未来大规模数据接入提供保障。   / 2.2 标识设备硬件的技术创新  /     不同于陆地动物，水产类动物生活在水中，对标识设备的设计要求更高。标识设备可以根据不同水产个体（如鱼、虾、蟹等）自动调整贴标方式，满足多样化需求。相比传统的人工标记设备，标识设备不仅能实现自动标记和溯源信息记录，还通过注入特定电子信号动态识别信息是否更新。标识信息会同步自动上传至云端数据库，避免人工录入可能产生的不可靠性或延迟。此外，设备的牢固性和防水性能经过特殊设计，能够适应水下环境且不易脱落。   / 2.3 追踪形式创新  /     标识设备所附的第一标签配备微型电子模块，在信息采集时同时记录最新的溯源信息与当前位置。通过底层电子范围辐射设计，当水产个体脱离规定养殖范围时，系统会触发自动报警，并通过北斗卫星精准定位其位置，及时通知相关。 / 2.4 防伪手段创新 / （1）标识设备操作系统基于多重用户身份验证，从而控制设备的开关机，防止非法登录。 / （2）溯源信息全自动采集，最大限度减少人工操作可能带来的信息错误。 / （3）当水产个体离开系统规定的养殖区域并进入运输或交易环节时，第一标签将更换为食品级的第二标签。通过养殖场与加工场订单贴标模式，实现水产来源与去向的绑定，确保实现“一鱼一码一市场”，形成可靠的全生命周期追溯链条。
